# Where Is Teemo

Cross-platform desktop app that lets you appear offline in League of Legends and VALORANT while still being able to play. Acts as a local proxy between the Riot client and Riot's chat servers, filtering XMPP presence stanzas.

## Architecture

The app runs as a system tray application (no dock icon). On launch:

1. **Config Proxy** — HTTP server on random port intercepts Riot client config requests from `clientconfig.rpg.riotgames.com`, rewrites `chat.host` to `127.0.0.1` and `chat.port` to 5223
2. **XMPP Proxy** — TLS server on `localhost:5223` accepts connections from Riot client using a locally-generated cert, forwards to real Riot chat server
3. **Riot Client** is launched with `--client-config-url` pointing to the config proxy
4. **Presence Filtering** — outgoing `<presence>` stanzas are rewritten to `type="unavailable"` when stealth mode is active; all other XMPP traffic passes unmodified

A locally-generated CA certificate is installed in the OS trust store so the Riot client accepts the TLS connection.

Communication between components uses `tokio::sync::watch` channels for stealth mode toggling, shutdown signals, and dynamic host updates.

## Tauri IPC Commands

Commands are defined in `src-tauri/src/commands.rs` with `#[tauri::command]` and registered in `lib.rs`. Frontend calls them via `invoke<T>("command_name", { args })`. Return types must derive `serde::Serialize`, error type is `String`.

| Command | Purpose |
|---------|---------|
| `get_status` | Current stealth mode, proxy status, connected game |
| `set_stealth_mode` | Toggle invisible/online (works mid-game) |
| `launch_game` | Full launch flow: kill existing, start proxies, launch Riot client |
| `stop_proxy` | Stop both proxies, reset state |
| `get_cert_status` | Check if CA/server certs exist and CA is trusted |
| `install_ca` | Install CA in OS trust store |
| `get_regions` / `set_region` | List/select from 16 supported regions |

## State Management

`AppState` wraps `Mutex<AppStateInner>` for thread-safe shared state. Access via `state.inner.lock().unwrap()`. Runtime channels (`mode_tx`, `shutdown_tx`, `host_tx`) are stored in state and used to control running proxies.

## Presence Stanza Handling

The stanza parser (`presence.rs`) handles TCP stream fragmentation with `find_stanza_end()`. **Key invariant:** child self-closing elements (e.g., `<pty/>` inside `<presence>`) must not trigger premature stanza splitting — only root-level `/>` terminates a self-closing tag.

When toggling modes mid-session, the proxy injects stanzas: unavailable presence for offline, or cached last presence for online.

## Key Gotchas

- The Riot client **must** be launched through the app (via `--client-config-url`) for the proxy to work
- Config proxy uses `danger_accept_invalid_certs(true)` because Riot's config endpoint may have cert issues
- `find_self_closing_end()` is quote-aware to avoid splitting on `/>` inside attribute values
- The app hides on close (intercepted via `CloseRequested`) — it never actually closes the window
- Certs are stored in the Tauri app data directory (`app.path().app_data_dir()`)
